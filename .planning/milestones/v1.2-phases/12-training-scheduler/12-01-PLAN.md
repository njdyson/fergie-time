<feature>
  <name>Training block computation and state types</name>
  <files>src/season/season.ts, src/season/training.ts, src/season/training.test.ts</files>
  <behavior>
    applyTrainingBlock behavior:
    - applyTrainingBlock(players, { 0: 'passing', 1: 'rest', 2: 'fitness' }) applies passing drill then fitness drill, skips rest day
    - Returns { updatedSquad: PlayerState[], deltas: Map<string, Partial<Record<keyof PlayerAttributes, number>>> }
    - deltas map contains accumulated gains per player across all drill days
    - Rest-only schedule ({ 0: 'rest', 1: 'rest', 2: 'rest' }) returns identical squad and zero-entry deltas
    - Empty schedule ({}) returns identical squad and zero-entry deltas
    - Multiple drills of same type accumulate gains correctly (diminishing returns apply)
    - Delta values are always >= 0 (no negative gains)

    DRILL_LABELS behavior:
    - DRILL_LABELS['fitness'] === 'Fitness'
    - Every DrillType key is present in DRILL_LABELS
    - Labels are title-cased human-readable strings

    TRAINING_DAYS_PER_MATCHDAY:
    - Constant equal to 3 (matches Phase 11 economy tuning assumption)

    SeasonState extension:
    - trainingSchedule? is optional (undefined on old saves)
    - trainingDeltas? is optional (undefined on old saves)
    - TrainingDayPlan type is DrillType | 'rest'
    - TrainingSchedule type is Record<number, TrainingDayPlan>
    - TrainingDeltas type is Map<string, Partial<Record<keyof PlayerAttributes, number>>>
  </behavior>
  <implementation>
    1. In src/season/season.ts: Add TrainingDayPlan, TrainingSchedule, TrainingDeltas type exports. Add trainingSchedule? and trainingDeltas? to SeasonState interface.

    2. In src/season/training.ts: Add TRAINING_DAYS_PER_MATCHDAY = 3 constant. Add DRILL_LABELS record mapping each DrillType to a title-cased label (e.g. 'fitness' -> 'Fitness', 'set_pieces' -> 'Set Pieces', 'oneOnOnes' would be 'One-on-Ones' if it existed, etc). Add applyTrainingBlock function:
       - Initialize deltas Map with empty object per player
       - Iterate over schedule entries sorted by day index
       - Skip 'rest' days
       - For each drill day, call existing applyDrill, compute per-attribute gains by diff, accumulate into deltas
       - Return { updatedSquad, deltas }

    3. In src/season/training.test.ts: Add new describe block for applyTrainingBlock with tests covering all behavior cases listed above.
  </implementation>
</feature>
