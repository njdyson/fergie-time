  <files>package.json, server/tsconfig.json</files>
  <action>
    Install production and dev dependencies:
    ```
    npm install express better-sqlite3 cookie-session
    npm install -D @types/better-sqlite3 @types/express @types/cookie-session tsx
    ```

    Create server/tsconfig.json with Node-targeted settings:
    - target: ES2022
    - module: ESNext
    - moduleResolution: bundler
    - strict: true
    - noEmit: true
    - skipLibCheck: true
    - types: ["node"]
    - erasableSyntaxOnly: true
    - include: ["."]

    IMPORTANT: Do NOT include verbatimModuleSyntax — better-sqlite3 is CJS and tsx handles the interop via default imports. The client tsconfig has verbatimModuleSyntax but the server must not.

    Add .gitignore entry for data/ directory (SQLite database files).
    Add dev:server script to package.json: "dev:server": "tsx watch server/index.ts"
  </action>
  <verify>
    npm ls express better-sqlite3 cookie-session tsx && cat server/tsconfig.json
  </verify>
  <done>All dependencies installed, server tsconfig exists with Node-appropriate settings, dev:server script in package.json</done>

<task type="auto">
  <name>Task 2: Express server with SQLite, health endpoint, and session middleware</name>
  <files>server/index.ts, server/db.ts, server/routes/health.ts</files>
  <action>
    Create server/db.ts:
    - Export getDb() function that returns a better-sqlite3 Database singleton
    - On first call: create data/ directory if missing (use fs.mkdirSync with recursive: true), open DB at data/fergie-time.db
    - Set pragma journal_mode = WAL
    - Set pragma foreign_keys = ON
    - Create saves table IF NOT EXISTS with columns: id INTEGER PRIMARY KEY AUTOINCREMENT, team_name TEXT NOT NULL UNIQUE, password_hash TEXT NOT NULL, game_state TEXT NOT NULL, version INTEGER NOT NULL DEFAULT 1, created_at TEXT NOT NULL DEFAULT (datetime('now')), updated_at TEXT NOT NULL DEFAULT (datetime('now'))
    - Export a closeDb() function for clean shutdown in tests

    Create server/routes/health.ts:
    - Export a Router (express.Router)
    - GET /api/health returns JSON: { status: 'ok', timestamp: new Date().toISOString() }

    Create server/index.ts:
    - Import express, cookie-session, getDb, healthRouter
    - Create Express app
    - app.use(express.json({ limit: '1mb' }))
    - app.use(cookieSession({ name: 'session', keys: ['fergie-time-dev-key'], maxAge: 24 * 60 * 60 * 1000 })) — dev key is fine for now, Phase 6 will use env var
    - app.use(healthRouter)
    - Call getDb() to initialize database on startup
    - Listen on process.env.PORT ?? 3001
    - Log "Server listening on port {PORT}" to console
    - Export app for testing

    IMPORTANT: Use default imports for express and better-sqlite3 (tsx handles CJS interop). Use `import type { ... }` for type-only imports per project convention.
  </action>
