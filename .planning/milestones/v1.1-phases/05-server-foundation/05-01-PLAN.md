  <behavior>
    TDD: Write tests first, then implement.

    Test cases for server/serialize.test.ts:

    1. "preserves Map types through JSON encode/decode" — Create a SeasonState-shaped object with a fatigueMap containing 2+ entries. Call serializeState(), then deserializeState(). Assert fatigueMap is instanceof Map, has correct size, and values match within floating point tolerance.

    2. "preserves version field" — Serialize state with version: 1. Deserialize. Assert version === 1.

    3. "handles empty Map" — fatigueMap with 0 entries round-trips to an empty Map (not undefined or {}).

    4. "handles full SeasonState from createSeason" — Import createSeason from src/season/season.ts. Create a real SeasonState. Serialize and deserialize. Assert fatigueMap.size > 0, all team names match, fixture count matches, table length matches, all scalar fields match. This catches any future Map fields.

    5. "serialized output is valid JSON string" — serializeState returns a string that JSON.parse can consume without throwing.

    6. "version field sits in envelope, not inside state" — The serialized JSON, when parsed raw, has a top-level "version" property.

    Implementation approach for server/serialize.ts:

    - Define a SaveEnvelope interface: { version: number; savedAt: string; state: SeasonState }
    - Use MAP_TAG = '__MAP__' sentinel pattern from research
    - serializeState(state: SeasonState, version?: number): string — wraps in envelope, uses JSON.stringify with replacer that converts Map instances to { __MAP__: true, entries: [...] }
    - deserializeState(json: string): SaveEnvelope — uses JSON.parse with reviver that reconstructs Map from tagged objects
    - Import SeasonState type from src/season/season.ts (pure interface, no DOM deps)
  </behavior>
