<feature>
  <name>Training drill system — applyDrill pure function</name>
  <files>src/season/training.ts, src/season/training.test.ts</files>
  <behavior>
    TRAIN-04 (drill → attribute improvement):
    - applyDrill([player], 'passing') returns player with higher passing, vision, crossing, distribution
    - applyDrill([player], 'fitness') returns player with higher pace, stamina, strength, acceleration, agility
    - Attributes NOT targeted by the drill remain unchanged
    - Output is a new PlayerState[] — original is not mutated

    TRAIN-04 (age factor):
    - A 19-year-old gains more per session than a 32-year-old (same attributes, same personality)
    - Age factor: ≤20 → 1.0, decays linearly through career, ≥35 → 0.15
    - Missing age (undefined) defaults to 25

    TRAIN-04 (personality / "training" trait):
    - Uses work_rate as the training personality proxy (per research decision — no new trait needed)
    - Training factor = 0.6 + work_rate * 0.8 → range [0.6, 1.4]
    - High work_rate (1.0) produces ~2.3x the gain of low work_rate (0.0)

    TRAIN-06 (no hidden ceiling, diminishing returns):
    - gain = BASE_DELTA * ageFactor * trainingFactor * (1 - currentValue)
    - A player at 0.50 improves faster than a player at 0.85
    - A player at 0.95 still improves, but gains are negligible
    - There is NO potential field, NO max cap — only the asymptotic (1 - currentValue) term

    Economy validation (headless 5-season sim):
    - Generate a weak-tier squad (base attrs ~0.45-0.65, all starting below 0.70)
    - Apply 114 drill sessions per season (38 matchweeks × 3 training days) for 5 seasons = 570 sessions
    - Cycle through all 8 drill types
    - Assert: no attribute on any player exceeds 0.95

    Edge cases:
    - Player with age undefined → treated as 25
    - Attribute at exactly 1.0 → gain is 0 (Math.min(1, ...) safeguard)
    - Empty player array → returns empty array
  </behavior>
