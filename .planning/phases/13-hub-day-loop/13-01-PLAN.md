```typescript
export interface DayDescriptor {
  dayIndex: number;      // 0-based (0,1,2 = training, 3 = match)
  label: string;         // 'Day 1', 'Day 2', 'Day 3', 'Match Day'
  type: 'training' | 'match';
  status: 'past' | 'current' | 'future';
}

export interface DayAdvanceResult {
  state: SeasonState;    // updated state with incremented currentDay
  isMatchDay: boolean;   // true if the NEW currentDay is match day
}
```

- `getDaySchedule(state: SeasonState): DayDescriptor[]` — Build array of TRAINING_DAYS_PER_MATCHDAY + 1 entries (3 training + 1 match). Mark each as past/current/future based on state.currentDay.

- `isMatchDay(state: SeasonState): boolean` — Returns `state.currentDay >= TRAINING_DAYS_PER_MATCHDAY`.

- `advanceDay(state: SeasonState): DayAdvanceResult` — Pure function:
  - If `isMatchDay(state)`, throw Error('Cannot advance past match day').
  - Read `state.trainingSchedule?.[state.currentDay]` (defaults to 'rest').
  - If it's a drill, call `applyDrill(playerTeam.squad, drill)` and compute per-player deltas by diffing attributes before/after. Merge deltas into existing `state.trainingDeltas` (additive — a player may have gained from a previous day's drill too).
  - If it's 'rest', no attribute changes.
  - Return `{ state: { ...state, currentDay: state.currentDay + 1, teams: updatedTeams, trainingDeltas: mergedDeltas }, isMatchDay: newDay >= TRAINING_DAYS_PER_MATCHDAY }`.

Key constraint: `advanceDay` must be a PURE function — it returns a new state, never mutates the input. This matches the project's functional state pattern (see `recordPlayerResult`, `simOneAIFixture`, `finalizeMatchday`).
