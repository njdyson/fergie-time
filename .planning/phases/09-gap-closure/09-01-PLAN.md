Two changes in src/main.ts:

1. **Wire onShirtNumberChange callback** — After the existing `onSelectionChange` block (around line 282), add:

```typescript
// Persist shirt number edits into season state (SQD2-03)
squadScreenViewInner.onShirtNumberChange((updatedPlayers: PlayerState[]) => {
  if (typeof seasonState !== 'undefined' && seasonState) {
    const playerTeam = seasonState.teams.find(t => t.isPlayerTeam);
    if (playerTeam) {
      playerTeam.squad = updatedPlayers;
    }
  }
});
```

This follows the exact same pattern as the `onSelectionChange` wiring directly above it. The callback fires when a user picks a shirt number from the popup. `getUpdatedPlayers()` creates new PlayerState objects via spread, so readonly fields are correctly handled. The updated squad array is written back to SeasonTeam.squad (which is mutable). Next auto-save will serialize the new numbers.

2. **Fix Hub kickoff stale stats** — At line 1517, change:
```typescript
squadScreenViewInner.update(playerTeam.squad, seasonState.fatigueMap, seasonState.squadSelectionMap);
```
to:
```typescript
squadScreenViewInner.update(playerTeam.squad, seasonState.fatigueMap, seasonState.squadSelectionMap, seasonState.playerSeasonStats);
```

This adds the missing 4th argument so G/A/App columns display current stats when entering squad via Hub Kick Off.
