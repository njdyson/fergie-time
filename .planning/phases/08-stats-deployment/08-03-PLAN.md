       - Draw shirt number centered on the shirt
       - Draw player initials below the shirt (first letter of first name + first letter of last name)
       - Keep it simple: colored rectangle with rounded top = shirt shape is fine
    3. **Player info panel** — grid showing:
       - Name, Age, Height (cm), Nationality, Position (role), Shirt Number
    4. **Attributes panel** — FM-style horizontal bars:
       - For each attribute in PlayerAttributes: pace, shooting, passing, dribbling, defending, physical
       - Label on left, bar on right, value 0.0-1.0 mapped to 0-100 display
       - Bar color: green (#4ade80) for >= 0.7, yellow (#fbbf24) for >= 0.4, red (#f87171) for < 0.4
       - Bar width proportional to value (max width = available space)
    5. **Personality panel** — same bar format for PersonalityVector fields:
       - directness, risk_appetite, composure, creativity, work_rate, aggression, anticipation, flair
       - Display names: Directness, Risk Appetite, Composure, Creativity, Work Rate, Aggression, Anticipation, Flair
    6. **Season stats panel** — table showing current season stats:
       - Season {N} heading
       - Stats: Apps, Goals, Assists, Shots, Shots on Target, Pass Completion %, Tackles Won, Yellow/Red Cards, Clean Sheets, Minutes Played
       - Show '-' for stats with zero appearances

    Dark theme using project palette. Scrollable if content exceeds viewport.

      - Show profile-screen div, hide others, hide nav tabs (profile is a full-page overlay)
    - Expose `showPlayerProfile` so screens can call it (either via a global function on window, or pass it as a callback when constructing screens)
    - Add 'profile-screen' div handling (add to index.html or create dynamically)
    - Back button in profile calls showScreen(previousScreen) to return
