  <files>src/ui/screens/playerProfileScreen.ts</files>
  <action>
In `playerProfileScreen.ts`:

1. **Delete the entire "Training Gains panel" block** (lines ~296-317) — the block starting with `// ── Training Gains panel (only if deltas exist for this player) ──` and ending with the closing `}` of the outer `if (trainingDeltas)`.

2. **Add a highlight indicator to attribute bars when that attribute has improved.** Modify the `renderBar` function to accept an optional `improved` boolean parameter:
   - Signature: `function renderBar(label: string, value: number, improved?: boolean): string`
   - When `improved` is true, add a small green upward arrow (▲) or a green dot next to the attribute value number, and give the bar a subtle green border or glow. Specifically:
     - Change the outer div to include a left border: `border-left: 3px solid ${GREEN}` when improved, else `border-left: 3px solid transparent`
     - Append a green `▲` after the value number: e.g. `<span style="color:${GREEN}; font-size:9px; margin-left:2px;">▲</span>`

3. **Pass deltas into the rendering logic.** In the `render()` method:
   - Before rendering coreAttrs, extract the player's deltas: `const playerDeltas = trainingDeltas?.get(player.id);`
   - When calling `renderBar` for each core attribute, check if the attribute has a positive delta. The core attrs map like this:
     - 'Pace' → `pace`, 'Shooting' → `shooting`, 'Passing' → `passing`, 'Dribbling' → `dribbling`, 'Defending' → `tackling`, 'Physical' → `strength`
   - Create a mapping from display label to attribute key to look up deltas. Or more simply, store tuples as `[label, value, attrKey]` and check `playerDeltas?.[attrKey] > 0`.
   - Also apply highlighting to the extended attributes (GK attrs) and render those bars with the improved flag too.

4. **Do NOT remove the `trainingDeltas` parameter** from `update()` or `render()` — it's still needed for the highlighting logic.
