<objective>
Wire halftime tactical changes and substitutions into the match lifecycle — the tactics board returns at halftime, the manager can reshuffle formation/duties and make up to 3 substitutions from a 5-player bench, and changes take effect at second half kickoff.

Purpose: Halftime changes are the core tactical lever in football management. Without them, the tactics board is a one-shot setup screen. This makes tactical decisions an ongoing part of match management.

Output: Complete halftime flow with formation changes and substitutions working end-to-end.
</objective>

  <action>
    1. **Expand squad to 16 players (11 starters + 5 bench):**
       - Modify `createMatchRosters()` to produce 16 players per team (extend with 5 more archetypes for the bench: e.g., "utilityDefender", "youngProspect", "targetMan", "playmaker", "speedster").
       - Add a `bench: PlayerState[]` field to MatchConfig or handle bench players as part of the roster but not on the pitch initially.
       - The engine should store bench players separately — they are not part of the 22-player simulation until substituted in.

    2. **Substitution engine method:**
       - Add `substitutePlayer(teamId: TeamId, outPlayerId: string, inPlayer: PlayerState): void` to SimulationEngine.
       - This replaces the outgoing player in the players array with the incoming bench player. The incoming player:
         - Gets the outgoing player's `formationAnchor` and `role` and `duty`
         - Gets their position set to the formation anchor (they "appear" at their tactical position)
         - Starts with `fatigue: 0` (fresh sub)
         - Retains their own `attributes` and `personality` (that's the whole point of subs)
       - Track substitution count per team (max 3 per match per TAC-05).
       - Add `getSubstitutionCount(teamId: TeamId): number` and `getBench(teamId: TeamId): PlayerState[]`.

    3. **Halftime detection and flow in main.ts:**
       - In the fullTimePoll interval (or a new halftimePoll), detect when `snap.matchPhase === MatchPhase.HALFTIME`.
       - When halftime detected:
         a. Pause the game loop (`setPaused(true)`)
         b. Switch to tactics view (`showTacticsView()`)
         c. Update tactics board to reflect current formation/roles/duties (in case they want to see what they started with)
         d. Show a "Start 2nd Half" button instead of "Kick Off"
       - When "Start 2nd Half" clicked:
         a. Read `tacticsBoard.getTacticalConfig()` for any formation/role/duty changes
         b. Apply pending substitutions from the tactics board
         c. Call `engine.setHomeTactics(config)` with the new config
         d. Switch to match view and unpause

    4. **Substitution UI on tactics board:**
       - Add a bench panel to the tactics board (below the pitch diagram or as a side panel):
         - Shows 5 bench player names/roles with their key attributes
         - To substitute: click a player dot on the pitch (selects them), then click a bench player. The bench player replaces the pitch player.
         - Show a visual swap: the substituted-out player moves to bench (grayed out, marked "OFF"), the sub moves onto the pitch at the same position.
         - Show substitution counter: "Subs: 0/3"
         - If 3 subs already made, disable further substitution interaction.
       - Add `getSubstitutions(): Array<{outId: string, inPlayer: PlayerState}>` to TacticsBoard.
       - Add `setBenchPlayers(bench: PlayerState[])` to TacticsBoard.
       - Add `setSubsRemaining(count: number)` to TacticsBoard.

    5. **In index.html:**
       - Add a bench panel container within `#tactics-screen`.
       - Style bench player rows: small text, monospace, dark theme, showing name, position, key stats.
       - Add "Start 2nd Half" button (hidden initially, shown at halftime).
       - Add substitution counter display.
  </action>
