<task type="auto">
  <name>Task 1: Tactics board canvas with draggable player dots and formation presets</name>
  <files>src/ui/tacticsBoard.ts, index.html</files>
  <action>
    1. Create `src/ui/tacticsBoard.ts` with a `TacticsBoard` class:
       - Constructor takes a canvas element and optional initial formation (defaults to '4-4-2')
       - Stores internal state: 11 player positions (Vec2[]), selected formation preset (FormationId), per-player duty (Duty[]), per-player role (Role[])
       - On construction, positions/roles loaded from FORMATION_TEMPLATES

    2. **Rendering (draw method):**
       - Draw a top-down pitch diagram similar to the match renderer (green background, white lines for outline, halfway, center circle, penalty areas). Use same pitch dimensions (105x68m) but map to the canvas with appropriate padding.
       - Draw 3 faint horizontal guide lines at zone boundaries (defensive ~33m, midfield ~55m lines from home perspective) with labels "DEF", "MID", "ATT".
       - Draw 11 player dots as filled circles (larger than match view, ~14px radius for easy clicking/dragging). Color: team blue (#3366cc). When a player has ATTACK duty, add a small upward arrow indicator. DEFEND duty: small shield indicator. SUPPORT: no indicator.
       - Show player role label below each dot (e.g., "ST", "CM", "LB").
       - Show player number (1-11) inside each dot.
       - If a player is selected (clicked), draw a highlight ring around them.

    3. **Drag-and-drop:**
       - On mousedown/touchstart: hit-test player dots. If hit, mark that player as "dragging".
       - On mousemove/touchmove: update dragging player's position (convert canvas coords to pitch coords). Clamp to pitch boundaries.
       - On mouseup/touchend: drop the player. Call `autoAssignRole()` to update their role based on new position. Store the new position.
       - The drag should feel responsive — redraw on every mousemove.

    4. **Formation preset picker:**
       - `setFormation(formationId: FormationId)` method: resets all 11 positions to the template's base positions and roles. Duties are preserved (not reset to Support on formation change — user might have set duties).
       - `getFormation(): FormationId` returns current formation.

    5. **Duty picker popup:**
       - On click (not drag) of a player dot: select that player and show a small popup near the dot with 3 buttons: "Defend", "Support", "Attack".
       - Clicking a duty button sets the player's duty and closes the popup.
       - Clicking elsewhere closes the popup.

    6. **Public API:**
       - `getTacticalConfig(): TacticalConfig` — returns the current formation, roles, and duties as the config the engine needs.
       - `show()` / `hide()` — controls visibility of the tactics board.
       - `render()` — draws the current state. Called on construction and after any change.

    7. In `index.html`:
       - Add a `<div id="tactics-screen">` wrapper containing:
         - A `<canvas id="tactics-canvas">` for the tactics board (same width as match canvas)
         - A formation preset selector: 5 buttons (`<button class="formation-btn" data-formation="4-4-2">4-4-2</button>` etc.)
         - A "Kick Off" button (reuse or duplicate the existing kickoff button concept)
       - The tactics screen is initially visible; the pitch-area (match view) is initially hidden.
       - Style the tactics screen dark theme consistent with existing UI (dark background, monospace font, blue accents).
       - Add the formation selector buttons row styled like the existing control bar.
  </action>
