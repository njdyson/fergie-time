  <action>
Create three files in `src/ui/portrait/`:

**1. `palettes.ts`** — Nationality palette mapping.
- Export `NationalityPalette` interface: `{ skin: readonly string[]; hair: readonly string[] }`.
- Export `NATIONALITY_PALETTES: Record<string, NationalityPalette>` with entries for all 10 nationality codes used in the project: GB, ES, FR, DE, BR, IT, PT, NL, AR, NG. Group by region:
  - Northern/Western Europe (GB, FR, NL, DE): fair to medium skin tones, blonde/brown/red/dark hair.
  - Southern Europe (ES, IT, PT): medium to olive skin, dark hair dominant.
  - South America (BR, AR): medium to brown skin, very dark hair.
  - West Africa (NG): deep brown to very dark skin, very dark hair.
- Export `FALLBACK_PALETTE` covering the full skin/hair range for unknown nationalities.
- Export `getPalette(nationality?: string): NationalityPalette` that returns the matching palette or fallback.
- Use the hex values from the research document as starting points.

**2. `portraitGenerator.ts`** — Seeded pixel art renderer.
- Import `createRng` from `../../simulation/math/random.ts` and `getPalette` from `./palettes.ts`.
- Constants: `GRID_W = 20`, `GRID_H = 24`, `SCALE_X = 6`, `SCALE_Y = 5` (20*6=120, 24*5=120). `BG_COLOUR = '#0f172a'`.
- Export `generatePortrait(canvas: HTMLCanvasElement, player: PlayerState): void`.
  - Seed RNG with `portrait-${player.id}` (namespaced to avoid collisions).
  - Select palette via `getPalette(player.nationality)`.
  - Fixed decision sequence (order must never change after shipping):
    1. `skinIdx` — index into palette.skin
    2. `hairStyleIdx` — index into HAIR_STYLES array
    3. `hairColIdx` — index into palette.hair
    4. `eyeColIdx` — index into EYE_COLOURS array
    5. `hasFacialHair` — boolean (rng() > 0.65)
  - Document the call order in a comment block at the top of the function.
  - Clear canvas with BG_COLOUR, then draw layers back-to-front:
    1. Hair back layer
    2. Skin (oval face)
    3. Eyes
    4. Mouth
    5. Facial hair (if applicable)
    6. Hair front layer
- Design the face template as coordinate arrays. The face should be a symmetric oval occupying roughly the center 10x14 pixels of the 20x24 grid, vertically centered with slight upward bias to leave room for neck/shoulders hint.
- Create at least 5 distinct hair styles (short crop, medium side part, long, shaved/buzz, curly/textured). Each style has a `back` and `front` coordinate array.
- `EYE_COLOURS`: at least 5 options (blue, green, brown, dark brown, hazel).
- Helper: `setPixel(ctx, gx, gy, colour)` draws a single logical pixel as a `fillRect(gx*SCALE_X, gy*SCALE_Y, SCALE_X, SCALE_Y)`.
- Helper: `drawLayer(ctx, pixels, colour)` iterates pixel array and calls setPixel.
- Add a simple ear indicator (1-2 pixels on each side of the head at eye level, slightly darker than skin).
- Add a subtle nose indicator (1-2 pixels below eye line, slightly darker than skin).

**3. `portraitCache.ts`** — Session-level cache.
- Module-level `const cache = new Map<string, ImageData>()`.
- Export `getOrGeneratePortrait(canvas: HTMLCanvasElement, player: PlayerState): void`.
  - Check cache for `player.id`. If hit, `ctx.putImageData(cached, 0, 0)` and return.
  - Otherwise call `generatePortrait(canvas, player)`, then `cache.set(player.id, ctx.getImageData(0, 0, canvas.width, canvas.height))`.
- Export `clearPortraitCache(): void` for testing.

**Important constraints:**
- Never use `Math.random()` — always `createRng`.
- Never use `canvas.toDataURL()` — use `getImageData`/`putImageData`.
- Keep canvas fixed at 120x120.
- The face template must be bilaterally symmetric (design left half, mirror right half in the coordinate data).
