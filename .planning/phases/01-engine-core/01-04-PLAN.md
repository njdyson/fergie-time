  <behavior>
    **Phase State Machine (advancePhase):**
    - KICKOFF at tick 0 → FIRST_HALF at tick 1 (after kickoff processed)
    - FIRST_HALF → HALFTIME at tick 2700 (half of 5400 total ticks for 90-min match at 30/sec)
    - HALFTIME → SECOND_HALF at tick 2701 (immediate transition)
    - SECOND_HALF → FULL_TIME at tick 5400
    - FULL_TIME stays FULL_TIME (terminal state)
    - Phase transitions produce MatchEvent entries (halftime, fulltime)
    - After a goal: phase briefly returns to KICKOFF for restart, then continues

    **Constants:**
    - TICKS_PER_HALF = 2700 (90 seconds * 30 ticks/sec / 2... actually 45 min * 60 sec * 30 ticks / 1 = 81000 ticks for a full match... but we compress match time)
    - Decision: Sim time compression. A "90-minute" match runs for 5400 ticks (3 real minutes at 30/sec). This is configurable. Each tick represents ~1 second of match time.
    - TOTAL_MATCH_TICKS = 5400 (configurable)

    **Goal Detection (checkGoal):**
    - Ball at x <= LEFT_GOAL_LINE (0) AND y within goal width AND z < CROSSBAR_HEIGHT → home concedes (away scores)
    - Ball at x >= RIGHT_GOAL_LINE (pitch width, e.g. 105) AND y within goal width AND z < CROSSBAR_HEIGHT → away concedes (home scores)
    - Ball at goal line but y outside goal width → no goal (ball goes out)
    - Ball at goal line but z >= CROSSBAR_HEIGHT → no goal (ball over bar)
    - Pitch dimensions: 105m x 68m. Goal width: 7.32m (centered on y). Crossbar height: 2.44m.

    **createInitialSnapshot:**
    - Returns SimSnapshot at tick 0, score [0,0], 22 players in kickoff positions, ball at center
    - Players positioned in a basic 4-4-2 for both teams (home and away)

    **applyGoal(snapshot, scoringTeam):**
    - Returns new SimSnapshot with incremented score and ball reset to center
  </behavior>
