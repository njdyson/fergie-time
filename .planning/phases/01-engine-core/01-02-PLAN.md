<feature>
  <name>2.5D Ball Physics</name>
  <files>src/simulation/physics/ball.ts, src/simulation/physics/ball.test.ts</files>
  <behavior>
    **Integration (integrateBall):**
    - Ball on ground (z=0) with velocity (10, 0) decelerates each tick via GROUND_FRICTION (0.985 per-tick multiplier)
    - After ~90 ticks (~3 seconds at 30fps), ball speed is < 0.5 m/s (effectively stopped)
    - Ball with positive vz rises, gravity (9.8 m/s^2) pulls it back down
    - Ball with z > 0 experiences NO ground friction on XY velocity
    - When z goes below 0: z clamped to 0, vz inverted and multiplied by BOUNCE_COEFFICIENT (0.55)
    - When |vz| < 0.1 after bounce: vz set to 0 (ball settles)
    - Ball position updates by velocity * dt each tick
    - Function is pure: takes BallState + dt, returns new BallState

    **Continuous Collision Detection (continuousCollisionCheck):**
    - Ball moving at (100, 0) m/s past a player at (5, 0) with radius 1.0: CCD detects collision
    - Ball moving at (1, 0) m/s with player at (50, 0): CCD returns false (doesn't reach)
    - Ball moving perpendicular to player (missing entirely): CCD returns false
    - Parametric t value must be in [0, 1] for valid collision (within this tick's movement)

    **Constants:**
    - GRAVITY = 9.8 m/s^2
    - GROUND_FRICTION = 0.985 (per-tick multiplier when on ground)
    - BOUNCE_COEFFICIENT = 0.55
    - SETTLE_THRESHOLD = 0.1 (vz below this after bounce → settle to 0)
  </behavior>
  <implementation>
Implement `integrateBall(ball: BallState, dt: number): BallState` following the research Pattern 7 code.

Key implementation details:
1. Convert dt from milliseconds to seconds: `dtSec = dt / 1000`
2. Update Z first: `vz -= GRAVITY * dtSec`, `z += ball.vz * dtSec` (use pre-update vz for position)
3. Ground clamp: if z <= 0, set z = 0, invert vz with bounce coefficient, settle check
4. Apply ground friction only when z === 0
5. Update position by velocity * dtSec
6. Return new BallState (immutable — never mutate input)

Implement `continuousCollisionCheck(ballPos: Vec2, ballVel: Vec2, playerPos: Vec2, dt: number, playerRadius: number): boolean` using parametric ray-circle intersection from research code. Discriminant check, t in [0, 1] range.

Export both functions and all constants (for testing).
  </implementation>
</feature>
