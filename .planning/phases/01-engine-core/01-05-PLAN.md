  <action>
Create `src/loop/gameLoop.ts` — fixed-timestep accumulator following research Pattern 1:
- `startGameLoop(engine: SimulationEngine, renderer: CanvasRenderer): void`
- `stopGameLoop(): void` (cancels requestAnimationFrame)
- FIXED_DT_MS = 1000/30 (33.33ms)
- Spiral-of-death guard: cap accumulated elapsed at 200ms
- Renderer receives `(prevSnapshot, currSnapshot, alpha)` for interpolation

Create `src/renderer/pitch.ts`:
- `drawPitch(ctx: CanvasRenderingContext2D, pitchToCanvas: (v: Vec2) => { x: number, y: number }): void`
- Draw pitch outline, center circle, center line, penalty areas, goal areas, penalty spots, corner arcs
- Use standard pitch dimensions: 105m x 68m
- Colors: green background (#2d8a4e), white lines (#ffffff), 2px line width

Create `src/renderer/canvas.ts` — the `CanvasRenderer` class:
- Constructor takes a `<canvas>` element. Sets canvas size to fill available width with correct aspect ratio (105:68).
- `draw(prev: SimSnapshot, curr: SimSnapshot, alpha: number): void`
- Coordinate mapping: simulation meters → canvas pixels. `pitchToCanvas(v: Vec2)` maps (0..105, 0..68) to canvas pixel coordinates.
- Draw order: (1) clear canvas, (2) pitch lines via drawPitch, (3) player circles, (4) ball shadow, (5) ball
- Player rendering: filled circles, 6px radius. Home team = blue (#3366cc), Away team = red (#cc3333). Direction indicator: small line from center in velocity direction.
- Ball rendering per research code: shadow at ground position (ellipse, alpha scales with z), ball circle scales with z: `radius * (1 + z/MAX_BALL_Z * 0.5)`. Ball color = white (#ffffff) with black outline.
- Round all coordinates to integers via Math.floor() to avoid sub-pixel rendering overhead.
- Interpolation: lerp player positions and ball position between prev and curr snapshots using alpha for smooth 60fps display.

Update `index.html`:
- Add a `<canvas id="match-canvas">` element
- Basic CSS: canvas centered, black background, no scroll

Update `src/main.ts`:
- Import SimulationEngine, CanvasRenderer, startGameLoop

- Create renderer attached to canvas element
- Call startGameLoop
- Give the ball an initial velocity so it moves visibly on load (e.g., kick toward goal at 15 m/s)
