This is the integration task. Update `src/simulation/engine.ts` to wire every subsystem built in Plans 01-09 into the tick() method.

**Engine tick() sequence (order matters):**

```
1. Advance match phase (phases.ts)
2. If HALFTIME or FULL_TIME → skip physics/AI, return snapshot
3. Accumulate fatigue for all players (fatigue.ts)
4. Compute effective attributes and personality for each player (fatigue attenuation + erosion)
5. Update spatial grid with current player positions (spatial.ts)
6. Compute formation anchors based on ball position and possession (formation.ts)
7. Build AgentContext for each player
8. For each player: run selectAction to get ActionIntent (agent.ts)
9. Log all agent decisions to DecisionLog (decisionLog.ts)
10. Resolve action intents:
    a. Movement: compute steering forces from intents (steering.ts), integrate positions
    b. Ball control: pass/shoot/dribble change ball velocity and carrier
    c. Contact: check tackles, shielding, aerials (contact.ts)
11. Integrate ball physics (ball.ts)
12. Check for goals (state.ts)
13. Accumulate match statistics (stats.ts)
14. Produce new SimSnapshot
```

**CRITICAL — Action intent resolution (step 10):**
All agents produce intents simultaneously. A separate integration step resolves them:
- MOVE_TO_POSITION / MAKE_RUN → steering seek/arrive toward intent target
- PRESS → steering pursuit toward ball carrier
- PASS_FORWARD / PASS_SAFE → set ball velocity toward target teammate, clear carrierId
- SHOOT → set ball velocity toward goal, clear carrierId
- DRIBBLE → small ball velocity in player's facing direction, maintain carrierId
- HOLD_SHIELD → no ball velocity change, maintain position
- Ball carrier detection: each tick, if ball is within CONTROL_RADIUS (1.5m) of a player and no carrier, that player becomes carrier

**Anti-swarming:** Apply formation anchor distance penalty as a multiplicative factor on ball-approaching actions (PRESS, MAKE_RUN, DRIBBLE when far from ball). This is already encoded in the action considerations (Plan 06) but verify it's working by checking spatial spread.

**Separation force:** Apply separation steering to all players to prevent overlapping. Separation radius = 2m. This runs AFTER intent-based movement as a post-processing correction.
