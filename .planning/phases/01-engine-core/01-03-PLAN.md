<feature>
  <name>Steering Behaviors</name>
  <files>src/simulation/physics/steering.ts, src/simulation/physics/steering.test.ts</files>
  <behavior>
    **seek(position, target, maxSpeed):**
    - Returns desired velocity pointing from position to target, magnitude = maxSpeed
    - seek(Vec2(0,0), Vec2(10,0), 5) → Vec2(5, 0)
    - seek(Vec2(0,0), Vec2(3,4), 10) → Vec2(6, 8) (normalized direction * maxSpeed)
    - When position === target, returns Vec2.zero()

    **arrive(position, target, maxSpeed, slowingRadius):**
    - Outside slowing radius: behaves like seek (full speed)
    - Inside slowing radius: speed proportional to distance/slowingRadius
    - At target (dist < 0.01): returns Vec2.zero()
    - arrive(Vec2(0,0), Vec2(100,0), 10, 20) → magnitude ~= 10 (outside slowing radius)
    - arrive(Vec2(0,0), Vec2(10,0), 10, 20) → magnitude ~= 5 (halfway through slowing radius)
    - arrive(Vec2(0,0), Vec2(0.005,0), 10, 20) → Vec2.zero() (at target)

    **separation(position, neighbors, radius):**
    - No neighbors within radius → Vec2.zero()
    - One neighbor at half radius → force pointing away, magnitude inversely proportional to distance
    - Multiple neighbors → summed repulsion forces
    - separation(Vec2(0,0), [Vec2(1,0)], 5) → force pointing in (-x) direction
    - Neighbors exactly at position (dist=0) are skipped (avoid division by zero)

    **pursuit(position, target, targetVelocity, maxSpeed):**
    - Predicts future target position by look-ahead proportional to distance
    - Stationary target (velocity=0) → same as seek
    - Moving target → seeks a position ahead of the target

    **clampVelocity(velocity, maxSpeed):**
    - If velocity.length() > maxSpeed, returns velocity normalized * maxSpeed
    - If velocity.length() <= maxSpeed, returns velocity unchanged
  </behavior>
  <implementation>
Implement each function following the research Pattern 6 code (Craig Reynolds 1999 model).

All functions are pure — they take Vec2 inputs and return Vec2 outputs. No player state mutation.

The `clampVelocity` function is the attribute cap: `maxSpeed = player.attributes.pace * BASE_SPEED * (1 - fatiguePenalty)`. The BASE_SPEED constant and fatigue calculation are NOT part of this plan — this plan only provides the raw steering functions. The caller (agent system in Plan 06/08) computes maxSpeed from attributes.

Export a `BASE_PLAYER_SPEED` constant (e.g., 8.0 m/s — an average footballer's sprint is ~8-9 m/s, cruising ~5-6 m/s) for use by callers. This is the speed a player with pace=1.0 and fatigue=0 achieves.
  </implementation>
</feature>
