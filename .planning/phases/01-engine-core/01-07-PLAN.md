<feature>
  <name>Fatigue System</name>
  <files>src/simulation/ai/fatigue.ts, src/simulation/ai/fatigue.test.ts</files>
  <behavior>
    **Fatigue Accumulation (accumulateFatigue):**
    - Takes current fatigue (0..1), tick number, stamina attribute (0..1), and dt
    - Gradual rate through first 60 match-minutes (3600 ticks at 1 tick = 1 match-second)
    - Steep rate in final 30 minutes (tick > 3600)
    - High stamina (0.9) → fatigue rate multiplied by ~0.6 (40% slower fatigue)
    - Low stamina (0.2) → fatigue rate multiplied by ~1.3 (30% faster fatigue)
    - At tick 0: fatigue = 0
    - At tick 2700 (halftime, 45min): fatigue ~0.15-0.25 depending on stamina
    - At tick 5400 (fulltime, 90min): fatigue ~0.5-0.8 depending on stamina
    - Fatigue clamped to [0, 1]
    - Work rate increases fatigue rate slightly (high work_rate players tire faster)

    **Attribute Attenuation (applyFatigueToAttributes):**
    - `effectiveAttribute = baseAttribute * (1 - fatigue * ATTENUATION_FACTOR)`
    - ATTENUATION_FACTOR ~0.5 (at fatigue=1.0, attributes reduced by 50%)
    - Affects pace, strength, stamina (physical attributes)
    - Does NOT affect technical attributes (passing, shooting, dribbling) — fatigue makes you slow, not unskilled
    - Actually: mild effect on technical attributes too (fatigue * 0.2 factor) — tired players miscontrol slightly
    - Returns new PlayerAttributes (immutable)

    **Personality Erosion (applyFatigueToPersonality):**
    - `effectiveTrait = lerp(baseTrait, conservativeDefault, fatigue * EROSION_FACTOR)`
    - EROSION_FACTOR ~0.6 (at fatigue=1.0, personality shifted 60% toward conservative)
    - Conservative defaults: { directness: 0.3, risk_appetite: 0.2, composure: 0.4, creativity: 0.3, work_rate: 0.3, aggression: 0.3, anticipation: 0.5, flair: 0.2 }
    - A fatigued maverick (directness=0.9) shifts toward directness=0.54 at full fatigue
    - A fatigued aggressive player (aggression=0.9) shifts toward aggression=0.54 at full fatigue
    - Returns new PersonalityVector (immutable)
    - Erosion is temporary (per-tick calculation from base values, not cumulative mutation)
  </behavior>
  <implementation>
Create `src/simulation/ai/fatigue.ts`:

- Export `CONSERVATIVE_DEFAULTS: PersonalityVector` constant
- Export `ATTENUATION_FACTOR = 0.5` and `EROSION_FACTOR = 0.6`

- `accumulateFatigue(currentFatigue: number, matchMinute: number, stamina: number, workRate: number): number`
  - matchMinute derived from tick: `matchMinute = tick / 60` (each tick = 1 match-second)
  - Gradual rate: `baseRate = 0.004` per match-minute (for average stamina)
  - Steep rate after minute 60: `baseRate = 0.012` per match-minute
  - Stamina modifier: `staminaMod = 1.5 - stamina` (stamina=1 → 0.5x rate, stamina=0 → 1.5x rate)
  - Work rate modifier: `workRateMod = 0.8 + workRate * 0.4` (work_rate=1 → 1.2x rate, work_rate=0 → 0.8x rate)
  - `newFatigue = currentFatigue + baseRate * staminaMod * workRateMod * (dt/1000)`
  - Clamp to [0, 1]

- `applyFatigueToAttributes(base: PlayerAttributes, fatigue: number): PlayerAttributes`
  - Physical: multiply by `(1 - fatigue * 0.5)`
  - Technical: multiply by `(1 - fatigue * 0.2)`

- `applyFatigueToPersonality(base: PersonalityVector, fatigue: number): PersonalityVector`
  - Each trait: `lerp(base[trait], CONSERVATIVE_DEFAULTS[trait], fatigue * EROSION_FACTOR)`

Write tests first. Key test: simulate full 90-minute match of fatigue accumulation for a high-stamina and low-stamina player, verify the curves diverge as expected.
  </implementation>
