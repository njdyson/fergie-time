<objective>
Build match statistics accumulation, the per-agent decision log with score range audit, and a click-to-inspect debug overlay.

Purpose: Statistics give the match quantitative meaning (possession %, shots). The decision log and audit tool are the primary calibration feedback mechanism — without them, utility score degeneracy is invisible. The debug overlay makes the simulation observable rather than a black box. The research is emphatic: build observability BEFORE the first full playable match.

Output: Stats accumulation from agent intents, decision ring buffer with audit function, and a Canvas debug overlay that shows action scores on player click.
</objective>

  <behavior>
    - DecisionLog stores entries per agent in a ring buffer (max 300 entries = 10 seconds at 30fps)
    - log(agentId, scores, selected) adds entry; when buffer full, oldest entry overwritten
    - getEntries(agentId) returns recent entries for that agent
    - getLatest(agentId) returns most recent entry
    - auditScoreRanges over 1000 entries: if PASS_SAFE selected 450/1000 times → flags as degenerate (>40%)
    - auditScoreRanges: if SHOOT selected 30/1000 times by forwards → OK (>5%); if 10/1000 → flags as miscalibrated (<5%)
    - Audit groups by role: forwards, midfielders, defenders have different action frequency expectations
    - clear() empties all logs (for match reset)
  </behavior>
  <action>
Create `src/simulation/ai/decisionLog.ts`:

```typescript
interface AgentDecisionEntry {
  tick: number;
  agentId: string;
  scores: Array<{ action: ActionType; score: number }>;
  selected: ActionType;
}

interface ScoreAuditReport {
  actionFrequencies: Record<ActionType, number>;  // 0..1 (percentage)
  degenerate: ActionType[];   // actions > 0.40 selection frequency
  underused: ActionType[];    // actions < 0.05 selection frequency
}
```

- `DecisionLog` class:
  - Private: `Map<string, AgentDecisionEntry[]>` with ring buffer per agent
  - `BUFFER_SIZE = 300`
  - `log(entry: AgentDecisionEntry): void` — adds to agent's ring buffer
  - `getEntries(agentId: string): readonly AgentDecisionEntry[]`
  - `getLatest(agentId: string): AgentDecisionEntry | undefined`
  - `clear(): void`

- `auditScoreRanges(entries: readonly AgentDecisionEntry[]): ScoreAuditReport`
  - Computes selection frequency per action type
  - Flags degenerate (>0.40) and underused (<0.05)
  - This is used post-match for calibration, not per-tick

- `draw(ctx, snapshot, pitchToCanvas)`:

      - Current fatigue level (bar)

      - Currently selected action (highlighted)
    - Panel: semi-transparent black background, white text, positioned to avoid going off-canvas
  - If no agent inspected: draw nothing (overlay is invisible)
- `isActive(): boolean` — returns whether an agent is being inspected
- Click on empty space or press Escape → deselect agent

The debug overlay is drawn AFTER the main renderer draw call. It reads from the DecisionLog but does not modify it.

Panel rendering details:
- Background: rgba(0, 0, 0, 0.85), rounded rectangle
- Text: white, 12px monospace font
- Action bars: colored proportional to score (green = high, red = low)
- Selected action bar has a distinct highlight (white border)
- Panel width: 180px, height: varies with content
