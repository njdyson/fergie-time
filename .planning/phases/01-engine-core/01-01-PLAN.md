```typescript
// Team identification
type TeamId = 'home' | 'away';

// Match phases (ENG-13)
enum MatchPhase { KICKOFF, FIRST_HALF, HALFTIME, SECOND_HALF, FULL_TIME }

// Action types for utility AI (ENG-03) — 7 distinct actions
enum ActionType {
  PASS_FORWARD, PASS_SAFE, DRIBBLE, SHOOT,
  HOLD_SHIELD, MOVE_TO_POSITION, PRESS, MAKE_RUN
}
// Note: 8 enum values but pass-forward/pass-safe count as one "pass" action conceptually.
// The research doc notes this: "requirements say 7 but 8 appear meaningful — consolidate pass or split press/intercept"
// Decision: keep 8 as separate action types. The "7 actions" requirement (ENG-03) is satisfied
// because the agent evaluates all entries and selects the best — the count is behavioral, not structural.

// Personality vector (ENG-04)
interface PersonalityVector {
  readonly directness: number;      // 0..1
  readonly risk_appetite: number;   // 0..1
  readonly composure: number;       // 0..1
  readonly creativity: number;      // 0..1
  readonly work_rate: number;       // 0..1
  readonly aggression: number;      // 0..1
  readonly anticipation: number;    // 0..1
  readonly flair: number;           // 0..1
}

// Physical/technical attributes
interface PlayerAttributes {
  readonly pace: number;            // 0..1
  readonly strength: number;        // 0..1
  readonly stamina: number;         // 0..1
  readonly dribbling: number;       // 0..1
  readonly passing: number;         // 0..1
  readonly shooting: number;        // 0..1
  readonly tackling: number;        // 0..1
  readonly aerial: number;          // 0..1
  readonly positioning: number;     // 0..1
}

// Personality weight matrix (ENG-04) — maps action type to trait weights
type PersonalityWeightMatrix = Record<ActionType, Partial<Record<keyof PersonalityVector, number>>>;

// Ball state (ENG-01)
interface BallState {
  readonly position: Vec2;     // ground X/Y
  readonly velocity: Vec2;     // ground velocity
  readonly z: number;          // height
  readonly vz: number;         // vertical velocity
  readonly carrierId: string | null; // player holding ball, or null
}

// Player state
interface PlayerState {
  readonly id: string;
  readonly teamId: TeamId;
  readonly position: Vec2;
  readonly velocity: Vec2;
  readonly attributes: PlayerAttributes;
  readonly personality: PersonalityVector;
  readonly fatigue: number;         // 0..1 (0=fresh, 1=exhausted)
  readonly role: string;            // positional role
  readonly formationAnchor: Vec2;   // tactical home position
}

// Match events
interface MatchEvent {
  readonly tick: number;
  readonly type: 'goal' | 'shot' | 'pass' | 'tackle' | 'foul' | 'save' | 'kickoff' | 'halftime' | 'fulltime';
  readonly playerId?: string;
  readonly teamId?: TeamId;
  readonly position?: Vec2;
  readonly data?: Record<string, unknown>;
}

// Match statistics (ENG-12)
interface MatchStats {
  readonly possession: readonly [number, number]; // home %, away %
  readonly shots: readonly [number, number];
  readonly passes: readonly [number, number];
  readonly tackles: readonly [number, number];
}

// Agent context — the world state visible to one agent (ENG-03)
interface AgentContext {
  readonly self: PlayerState;
  readonly teammates: readonly PlayerState[];
  readonly opponents: readonly PlayerState[];
  readonly ball: BallState;
  readonly matchPhase: MatchPhase;
  readonly score: readonly [number, number];
  readonly distanceToOpponentGoal: number;
  readonly distanceToBall: number;
  readonly distanceToFormationAnchor: number;
  readonly isInPossessionTeam: boolean;
  readonly nearestDefenderDistance: number;
  readonly nearestTeammateDistance: number;
}

// Action intent — produced by agent, consumed by engine integration step
interface ActionIntent {
  readonly agentId: string;
  readonly action: ActionType;
  readonly target?: Vec2;           // where to pass/move/shoot
  readonly targetPlayerId?: string; // who to pass to
}

// The immutable simulation snapshot (Pattern 2 from research)
interface SimSnapshot {
  readonly tick: number;
  readonly timestamp: number;
  readonly ball: BallState;
  readonly players: readonly PlayerState[];
  readonly matchPhase: MatchPhase;
  readonly score: readonly [number, number];
  readonly events: readonly MatchEvent[];
  readonly stats: MatchStats;
}
```

All interfaces use `readonly` properties for immutability. All numeric ranges are 0..1 unless otherwise noted.

Create `src/simulation/math/vec2.ts` — immutable Vec2 class per the research code example. Methods: add, subtract, scale, length, normalize, dot, distanceTo, static zero(). The normalize method returns Vec2.zero() when length < 0.001 to avoid division by zero.

Write tests first (RED), then implement (GREEN).

  <files>src/simulation/math/random.ts, src/simulation/math/random.test.ts, src/simulation/math/curves.ts, src/simulation/math/curves.test.ts</files>
  <behavior>
    - createRng("seed1") produces deterministic sequence: calling rng() twice with same seed gives same values
    - createRng("seed1") !== createRng("seed2") — different seeds give different sequences
    - gaussianNoise(0, 1, rng) returns a finite number
    - gaussianNoise with stdDev=0 always returns mean
    - Over 1000 samples, gaussianNoise(0, 1) has mean near 0 (|mean| < 0.1) and stdDev near 1
    - sigmoid(0.5, 10, 0.5) ~== 0.5 (midpoint)
    - sigmoid(0, 10, 0.5) is close to 0
    - sigmoid(1, 10, 0.5) is close to 1
    - exponentialDecay(0, k) === 1 for any k
    - exponentialDecay(large, k) approaches 0
    - logarithmic(0, max) === 0
    - logarithmic(max, max) === 1
    - step(below_threshold) === 0, step(above_threshold) === 1
    - linear(x, m, c) clamps output to [0..1]
    - All curve functions return values in [0..1] range
  </behavior>
  <action>
Create `src/simulation/math/random.ts`:
- `createRng(seed: string): () => number` — wraps seedrandom to return a function producing [0..1) values
- `gaussianNoise(mean: number, stdDev: number, rng: () => number): number` — Box-Muller transform. If stdDev === 0, return mean immediately. Use `Math.max(rng(), 1e-10)` to avoid log(0).

Create `src/simulation/math/curves.ts`:
- `sigmoid(x: number, steepness: number, midpoint: number): number` — logistic curve, clamped to [0..1]
- `exponentialDecay(x: number, k: number): number` — `e^(-k*x)`, clamped to [0..1]
- `logarithmic(x: number, max: number): number` — `log(1+x)/log(1+max)`, returns 0 when max is 0
- `step(x: number, threshold: number): number` — returns 0 below threshold, 1 at or above
- `linear(x: number, slope: number, intercept: number): number` — `clamp(slope*x + intercept, 0, 1)`

All curve functions MUST return values clamped to [0..1]. This is a hard contract for the utility AI system.

Write tests first (RED), then implement (GREEN).
